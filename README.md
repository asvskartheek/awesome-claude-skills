## Skills Overview

### TF-IDF Search Engine (`/tfidf-search`)
Implements text search functionality using TF-IDF vectorization and cosine similarity. Use this skill when you need to build search engines for text datasets, find similar documents, or rank content by relevance.  Perfect for analyzing song lyrics, articles, or any text corpus.

### Kartheek's Skill Creator (`/kartheeks-skill-creator`)
A meta-skill that helps you create new Claude skills following best practices. Use this when you want to generate a new skill with proper structure, frontmatter, and documentation. It guides you through the entire skill creation workflow from requirements gathering to implementation.

### Integrating Clerk with Next.js (`/integrating-clerk-nextjs`)
Enforces current best practices for integrating Clerk authentication into Next.js App Router applications. Use this skill when adding Clerk authentication to your Next.js projects, ensuring you follow the latest patterns with `clerkMiddleware()` and proper setup.

### Writing Rap Lyrics (`/writing-rap-lyrics`)
Master rhythm and poetry to create professional rap lyrics with proper flow, structure, and timing. This skill teaches musical fundamentals (bars, beats, tempo, BPM) and provides table-based formatting for precise lyric timing.  Use when writing rap verses, understanding cadences, or improving your flow patterns.

### Training APO Agents (`/training-apo-agents`)
Train LangChain/LangGraph agents using Microsoft Agent-Lightning's APO (Automatic Prompt Optimization) algorithm. Use this skill when you need to optimize agent prompts automatically through reinforcement learning. Works with both single agents and multi-agent swarms, providing a complete workflow from prerequisites check to full training with smoke tests.

## Contributing

Contributions are welcome! Please read the [contribution guidelines](CONTRIBUTING.md) first.

We're looking for: 
- New Claude Skills (with SKILL.md files)
- MCP Server implementations
- Tutorials and guides
- Tools and libraries
- Bug fixes and improvements

Please ensure your pull request adheres to the following guidelines:
- Search previous suggestions before making a new one
- Make an individual pull request for each suggestion
- Use the following format:  `[Name](link) - Description.`
- New categories or improvements to the existing categorization are welcome
- Keep descriptions short and simple, but descriptive
- End all descriptions with a period
- Check your spelling and grammar
- Make sure your text editor is set to remove trailing whitespace

Thank you for your suggestions!
