# Contribution Guidelines

Thank you for considering contributing to Awesome Claude Skills! This document outlines the process and guidelines for contributions.

## What Qualifies as "Awesome"?

Before submitting a resource, please ensure it meets the following criteria:

- **Quality**: The resource should be high-quality, well-documented, and actively maintained
- **Relevance**: It should be directly related to Claude Skills or MCP (Model Context Protocol)
- **Uniqueness**: It should offer something unique or significantly better than existing alternatives
- **Accessibility**: It should be publicly accessible and usable by the community
- **Documentation**: It should have clear documentation or instructions for use

## How to Contribute

### Adding a New Resource

1. **Fork the repository** and create a new branch for your contribution
2. **Add your resource** to the appropriate section in the README.md
3. **Follow the format**: `[Name](link) - Description.`
4. **Keep descriptions concise**: One or two sentences maximum
5. **Maintain alphabetical order** within sections when possible
6. **Test the link** to ensure it's working and points to the correct resource
7. **Submit a pull request** with a clear title and description

### Creating a New Category

If you believe a new category is needed:

1. Ensure there are at least 3-4 high-quality resources for the category
2. Place the new category in the most logical position in the table of contents
3. Provide a brief description of what belongs in the category
4. Include the initial resources in alphabetical order

### Improving Existing Content

- Fix broken links
- Update outdated information
- Improve descriptions for clarity
- Reorganize content for better navigation
- Fix typos and grammar

## Pull Request Guidelines

Your pull request should:

- **Have a clear title** describing what you're adding or changing
- **Include one logical change** per pull request (add one resource, fix multiple typos, etc.)
- **Provide context** in the PR description explaining why the addition is valuable
- **Pass all checks** including link validation and formatting
- **Follow the existing style** and structure of the list

## Quality Standards

### For Claude Skills

- Must include a `SKILL.md` file or clear documentation
- Should demonstrate a specific, useful capability
- Must be well-structured and follow Claude Skills conventions
- Should include examples or usage instructions

### For MCP Servers

- Must implement the MCP specification correctly
- Should include installation and setup instructions
- Must have clear documentation of available tools/resources
- Should include examples of usage

### For Tutorials and Guides

- Must be technically accurate and up-to-date
- Should include step-by-step instructions or clear explanations
- Must be well-written and easy to follow
- Should include code examples where applicable

## Suggesting Removals

If you find a resource that:

- Has a broken or dead link
- Is no longer maintained
- Has been superseded by a better alternative
- Doesn't meet quality standards

Please open an issue explaining why it should be removed, or submit a pull request removing it with justification.

## Code of Conduct

This project follows the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to the project maintainers.

## Questions?

If you have questions about contributing, feel free to:

- Open an issue for discussion
- Reach out to the maintainers
- Start a discussion in the community channels
